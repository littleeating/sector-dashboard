# 板块动量看板设计

## 背景

当前项目是一个 Python 股票筛选脚本仓库，已有同花顺导出文件读取、规则筛选和 Excel 输出能力。本次新增目标是独立构建一个每天可更新的网页，用来展示从最近可用交易日开始回看 5、10、20、30、45、60 个交易日时，累计涨幅最大的股票板块，并把重点板块的上涨趋势放在同一张图表中对比。

该功能不改动现有 `stock_filter.py` 的筛选流程，新增为“板块动量看板”模块。

## 目标

1. 每天生成一个本地网页，默认输出到 `output/sector_dashboard/index.html`。
2. 统计行业板块和概念板块，二者分开展示，不混合排名。
3. 对每类板块分别计算 5、10、20、30、45、60 个交易日累计涨幅。
4. 每个周期展示累计涨幅最高的前 10 个板块。
5. 在同一张趋势图里绘制重点板块的累计涨幅曲线，便于比较不同板块的上涨持续性。
6. 页面显示最近数据日期、生成时间、数据源状态和失败提示。
7. 数据源失败时尽量使用本地缓存生成页面，避免网页空白。

## 非目标

1. 第一版不实现登录开盘啦、淘股吧 App 或网页抓取。
2. 第一版不做投资建议、买卖信号或自动交易。
3. 第一版不把行业板块和概念板块混成一个总榜。
4. 第一版不需要常驻 Web 服务，生成静态 HTML 即可。

## 数据源

主数据源采用 AKShare 封装的公开板块行情接口：

- 东方财富行业板块实时行情，用于获取行业板块清单和当前状态。
- 东方财富行业板块历史日频行情，用于计算行业板块历史累计涨幅。
- 东方财富概念板块清单和历史日频行情，用于计算概念板块历史累计涨幅。
- 同花顺行业或概念板块指数日频行情作为后续可选补充源。
- 雪球热度数据作为后续可选辅助字段，不作为第一版涨幅主数据。

开盘啦和淘股吧更偏短线情绪、题材讨论和涨停生态。第一版保留扩展接口位置，但不把它们作为必要数据源，否则每日自动更新会受登录、反爬和页面结构变化影响。

## 计算口径

“当日”定义为数据源返回的最近一个可用交易日。如果当天是非交易日，或数据源尚未更新到当天收盘数据，则使用最近一个有收盘价的交易日。

每个周期的累计涨幅按收盘价计算：

```text
累计涨幅 = 当前收盘价 / N 个交易日前收盘价 - 1
```

输出时转为百分比，保留两位小数。若某个板块历史数据不足对应周期，则该板块不参与该周期排名，并在数据质量统计中计入“历史不足”。

## 页面设计

页面是一个数据看板，不做营销页。布局保持紧凑、清晰、适合每天打开快速扫描。

顶部区域显示：

- 页面标题：板块动量看板
- 最近数据日期
- 页面生成时间
- 行业板块数量、概念板块数量
- 数据源状态摘要

主体区域包含：

- “行业板块”排行榜：按 5、10、20、30、45、60 个交易日分栏或分卡展示前 10 名。
- “概念板块”排行榜：同样按周期展示前 10 名。
- “重点趋势对比”图表：将各周期榜首去重后的板块放在同一张折线图里，展示最近 60 个交易日的累计涨幅走势。
- “数据源与异常”区域：列出成功、失败、缓存命中和历史不足情况。

视觉风格以白底、深色文本、克制的红绿涨跌色和清晰表格为主。排行榜和图表要优先保证数字可读性，避免装饰性元素干扰判断。

## 架构

新增模块建议拆分为四层：

1. `sector_data.py`
   负责调用 AKShare 数据源，拉取板块清单和历史日频数据，并把不同来源字段统一成标准结构。

2. `sector_momentum.py`
   负责计算各周期累计涨幅、排名、趋势图所需序列，以及数据质量统计。

3. `sector_dashboard.py`
   命令行入口。负责编排数据拉取、缓存读取/写入、计算、页面渲染和错误汇总。

4. `templates/sector_dashboard.html`
   静态网页模板。内嵌必要 CSS、排行榜数据和 SVG 趋势图，生成后可以直接用浏览器打开。

缓存目录建议使用 `cache/sector_dashboard/`，保存标准化后的板块历史数据和最近一次成功计算结果。输出目录建议使用 `output/sector_dashboard/`。

## 数据流

1. 命令行启动 `sector_dashboard.py`。
2. 程序读取配置，确定周期、前 N 名、输出路径和缓存路径。
3. 从 AKShare 获取行业板块和概念板块清单。
4. 对每个板块拉取足够覆盖 60 个交易日的历史日频数据。
5. 将成功数据写入缓存。
6. 如果某个源失败，尝试读取该板块最近缓存。
7. 计算各周期累计涨幅和排名。
8. 生成趋势图数据。
9. 渲染静态 HTML。
10. 在终端输出生成结果、数据日期、成功数量和失败数量。

## 命令行

第一版提供一个简单命令：

```powershell
& 'C:\Users\AERO\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe' sector_dashboard.py --output output/sector_dashboard/index.html
```

可选参数：

- `--top-n 10`
- `--periods 5,10,20,30,45,60`
- `--cache-dir cache/sector_dashboard`
- `--source eastmoney`
- `--max-workers 4`

## 每日更新

第一版提供可手动运行的脚本和 README 命令。每天自动更新通过 Windows 任务计划程序配置：

- 建议时间：交易日 16:30 之后。
- 操作：运行 Codex 内置 Python 执行 `sector_dashboard.py`。
- 输出：覆盖 `output/sector_dashboard/index.html`。

项目内可后续增加 `scripts/register_sector_dashboard_task.ps1` 来自动创建计划任务，但第一版先提供明确命令，避免误改用户系统计划任务。

## 错误处理

程序不得因为单个板块失败而整体失败。错误处理策略：

- 板块清单获取失败：尝试使用缓存清单；无缓存则终止并给出明确错误。
- 单个板块历史行情失败：记录失败并尝试使用该板块缓存。
- 历史数据不足：跳过对应周期排名，保留该板块可用周期。
- 全部数据不可用：不生成误导性排名，输出错误页面说明原因。

## 测试策略

单元测试重点覆盖：

- 累计涨幅计算。
- 历史数据不足时的跳过逻辑。
- 多周期排名。
- 趋势图序列生成。
- 数据源失败时缓存回退。
- HTML 渲染包含关键字段和图表数据。

外部数据源测试不依赖真实网络接口，使用固定 DataFrame 或 JSON fixture。真实 AKShare 拉取作为手动验证步骤。

## 依赖

当前 Codex 内置 Python 已有 `pandas`，但未安装 `akshare`、`jinja2`、`plotly`。实现时优先选择：

- `akshare`：获取板块数据。
- `jinja2`：渲染 HTML 模板。
- 内嵌 SVG：绘制趋势图，避免生成后的本地网页依赖 CDN 或外部 JavaScript。

若网络安装依赖不可用，页面渲染可退化为 Python 字符串模板。数据源仍需要 `akshare` 或用户提供导出数据。
